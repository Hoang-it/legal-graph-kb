## Notes

- HyDE1 records are produced fresh in this experiment (skip-when-on-disk).
  HyDE1 LLM calls hit the shared HyDE cache at `artifacts/hyde/openai__gpt-4o-mini/` populated by exp 08, so the incremental LLM cost for HyDE1 here is $0 if exp 08 already ran.
- HyDE2 has its own cache at `artifacts/hyde2/openai__gpt-4o-mini/` keyed by seed_clause_ids; the cache invalidates automatically when the LoRA model or the dense index changes (because seed sets shift).
- All three arms share encoder + index; only the dense query embedding differs (raw question / HyDE1 doc / HyDE2 doc).
- This is a retrieval-only diagnostic at article granularity. The thesis-defining E2E metric (academic_v2 strict tuple) lives in item B of `docs/plans/exp08_followups_and_strict_metric.md`.
