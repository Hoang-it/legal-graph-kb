## Notes

- All three arms share encoder + index with exp 09; only the dense query embedding differs (raw question / HyDE1-GRACE-R doc / HyDE2-GRACE-R doc).
- HyDE1 + HyDE2 cache dirs are shared with exp 08/09 (`artifacts/hyde/`, `artifacts/hyde2/`) but the new prompt_sha gives disjoint cache keys → exp 10 always starts cold for HyDE1+HyDE2.
- The `dense` arm has no LLM call; its records should match exp 09's `dense` records bit-for-bit (same retriever, same seed_k=0 for raw).
