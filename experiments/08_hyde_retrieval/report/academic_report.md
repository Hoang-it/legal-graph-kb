- **R-Precision** = precision at K=|gold| per question. Since K=|gold|, P = R = F1 — fair single number when |gold| << K_max.
- **MRR** = mean of `1/rank(first gold article)`. 0 if no gold retrieved.
- **NDCG@K** = binary-relevance, log2-discounted, normalised by ideal DCG of min(|gold|, K) ones.

## HyDE success criteria (plan §Success criteria)

HyDE wins if **any one** of the following holds on the in_corpus stratum:

1. `dense_hyde` lifts R@12 by ≥ **+3pp absolute** vs `dense`.
2. `dense_hyde` lifts NDCG@10 by ≥ **+5% relative** vs `dense`.
   (Approximated below with NDCG@12 since K=10 is not in the K-set.)
3. `full_rerank_hyde` lifts R-Precision by ≥ **+15% relative** vs `full_rerank`.

| criterion | metric | dense* | dense_hyde* / full_rerank_hyde* | Δ | threshold | passes? |
|---|---|---:|---:|---:|---:|:-:|
| 1 | R@12 in_corpus (abs Δ) | 0.3832 | 0.4736 | 0.0904 | +0.03 | ✓ |
| 2 | NDCG@12 in_corpus (rel Δ) | 0.2186 | 0.2944 | 0.3468 | +0.05 | ✓ |
| 3 | R-Precision in_corpus (rel Δ) | 0.1207 | 0.1200 | -0.0058 | +0.15 | — |

## Notes

- All 4 arms share encoder + index + reranker; only the dense query embedding differs (raw question vs HyDE doc).
- Recall denominator = |gold|; questions with empty gold skipped.
- Precision denominator = |retrieved@K|; empty retrieved set → precision = 0.
- NDCG normalised against the ideal DCG given |gold| relevant items.
- For per-stage funnel of `full_rerank_hyde`, run `python scripts/exp08_funnel.py`.
